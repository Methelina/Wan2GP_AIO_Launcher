# Wan2GP Launcher (UV)

**Portable, self-contained installer and launcher for Wan2GP on Windows.**

> This launcher uses **UV** — a fast, lightweight Python package manager — to create an isolated environment. No system‑wide Python or Conda installation is required. Everything stays inside the project folder.

---

## Особенности

- **Установка в один клик** – автоматически загружает UV, создаёт виртуальное окружение, устанавливает PyTorch и все зависимости.
- **Выбор GPU** – поддерживаются GTX 10xx (CUDA 12.8) и RTX 20xx‑50xx (CUDA 13.1).
- **Все ускорители включены** – SageAttention, FlashAttention, SpargeAttention, GGUF, Nunchaku, Bitsandbytes, Triton – устанавливаются автоматически.
- **Интерактивное меню запуска** – быстро переключайтесь между режимами:
  - Basic GUI (выбор модели из списка)
  - Image‑to‑Video (`--i2v`)
  - T2V 1.3B (`--t2v-1-3B`) – быстрый
  - T2V 14B (`--t2v-14B`) – высокое качество
- **Автоматический подбор порта** – если порт 7860 занят, скрипт найдёт следующий свободный.
- **Полная портативность** – все кеши, виртуальное окружение и бинарники UV находятся внутри папки со скриптом. Папку можно перемещать куда угодно.
- **Всегда использует `--listen --share`** – обеспечивает удалённый доступ и создаёт публичную ссылку Gradio при необходимости.

---

### Требования

- Windows 7 / 10 / 11 (x64)
- [Git](https://git-scm.com/download/win) установлен и доступен в `PATH`
- Интернет (для загрузки UV, Wan2GP и пакетов Python)
- Видеокарта NVIDIA с соответствующими драйверами (CUDA 12.8 или 13.1)

> Лаунчер **не требует** прав администратора.

---

### Установка и запуск

1. **Скачайте** файл `Wan2GP-Launcher.ps1` в пустую папку.
2. **Нажмите правой кнопкой** мыши на файл и выберите **«Запуск с помощью PowerShell»** – или откройте PowerShell в этой папке и выполните:
   
   ```powershell
   .\Wan2GP-Launcher.ps1
   ```
3. **При первом запуске** автоматически:
   - Загрузится UV (≈10 МБ).
   - Склонируется репозиторий Wan2GP.
   - Создастся виртуальное окружение Python.
   - Установится PyTorch (выберите тип GPU).
   - Установятся все зависимости и ускорители.
4. После установки появится **меню запуска** – выберите нужный режим.
5. Приложение откроется в браузере (или покажет ссылку Gradio, если localhost недоступен).

---

### Режимы запуска

| Опция           | Описание                                                       |
| --------------- | -------------------------------------------------------------- |
| **1. Basic**    | Стандартный GUI – выберите любую модель из выпадающего списка. |
| **2. I2V**      | Режим «изображение в видео» (`--i2v`).                         |
| **3. T2V 1.3B** | Более быстрая модель текст‑в‑видео (`--t2v-1-3B`).             |
| **4. T2V 14B**  | Более крупная и качественная модель (`--t2v-14B`).             |
| **5. Выход**    | Закрыть лаунчер.                                               |

---

### Как это работает

- Скрипт проверяет наличие Git и загружает `uv.exe` в папку `Bin\`.
- Создаёт виртуальное окружение Python в `Wan2GP\env\`.
- Устанавливает PyTorch с выбранной версией CUDA.
- Обрабатывает `requirements.txt` (с дополнительными индексами для `onnxruntime-gpu`).
- Устанавливает все колёса ускорителей по заданным URL.
- Затем показывает меню запуска.
- При каждом запуске ищет свободный порт (начиная с 7860) и передаёт `--server-port` в Wan2GP.

### Структура папок (после установки)

```
.
├── Bin\                 # Исполняемые файлы UV
├── Wan2GP\              # Клонированный репозиторий Wan2GP
│   ├── env\             # Виртуальное окружение Python
│   └── wgp.py           # Основное приложение
├── .wan2gp_config       # Сохранённая конфигурация GPU (создаётся автоматически)
└── Wan2GP-Launcher.ps1  # Этот скрипт
```

Все кеши (`uv`, `pip`, Hugging Face) хранятся внутри папки со скриптом – ничего не пишется в `%USERPROFILE%`.

---

### Возможные проблемы и решения

- **Порт уже занят** – лаунчер автоматически пробует следующий порт (7861, 7862, …).
- **Ошибка установки `onnxruntime-gpu`** – скрипт использует `--index-strategy unsafe-best-match` и правильный дополнительный индекс, поэтому зависимость должна разрешиться корректно.
- **Git не найден** – установите Git и добавьте его в `PATH`.
- **Отказано в доступе** – запускайте скрипт от обычного пользователя, права администратора не нужны.
